# 测试与验收说明

本文档与 [smoke_test_models.py](../smoke_test_models.py) 的 `--offline` 套件对齐，描述**一期默认主链路**的分层测试策略、模块用例矩阵及手工验收要点。

说明：仓库中仍保留审核、模板视觉、批量表格、多模态和 MiMo 等增强/实验代码，但它们**不再属于一期默认验收门禁**。以 [config.py](../config.py) 与 `app.py` 的当前主流程为准。

## 一、测试分层

| 层级 | 命令 / 方式 | 依赖 | 目的 |
|------|-------------|------|------|
| L0 离线 | `python smoke_test_models.py --offline` | 无 API Key | 一期主链路纯逻辑：路由、检索扩展、字数上限、WordFiller 清洗与锚点/占位相关离线回归 |
| L1 API 冒烟 | `python smoke_test_models.py`（可加 `--skip-vision`、`--skip-chroma`、`--skip-embed-api`） | 有效 Key | 小/大模型、视觉、Embedding、Chroma 入库、流式；并再次跑完整 L0 套件 |
| L1 模型矩阵 | `python smoke_test_models.py --probe-models` | 网关 Key | 13 模型 chat/vision/enable_search → `data/probe_gateway_models.md` |
| L1 评分榜 | `python scripts/run_leaderboard.py --quick` 或 `python smoke_test_models.py --leaderboard --leaderboard-quick` | 网关 + 百炼 Key | V/A/C/G + 速度 → `docs/模型评分榜.md`、`data/leaderboard/model_selection.yaml` |
| L2 端到端 | `streamlit run app.py`，实际上传模板与知识库并生成 | Key + 本地数据 | UI、Word 输出、侧栏开关与日志是否符合预期 |

## 二、L1：模型评分榜（V/A/C/G + 速度）

配置：[data/leaderboard/leaderboard_v2026.05.21.yaml](../data/leaderboard/leaderboard_v2026.05.21.yaml)

```bash
# 骨架自检（不调 API）
python scripts/run_leaderboard.py --dry-run

# 推荐：quick 子集 × 复星网关 + 百炼双通道
python scripts/run_leaderboard.py --quick --channels fosun,dashscope

# 指定模型
python scripts/run_leaderboard.py --models qwen3.5-plus,gpt-5.4 --channels fosun,dashscope

# 仅 C 榜离线（smoke --offline 映射到 repo_level）
python scripts/run_leaderboard.py --offline-c-only

# 经冒烟入口
python smoke_test_models.py --leaderboard --leaderboard-quick
```

**产出**：

- `docs/模型评分榜.md` — 质量主榜、G 榜、低延迟副榜、E 榜
- `data/leaderboard/results_<timestamp>.json` — 原始分数
- `data/leaderboard/model_selection.yaml` — 各 `config.py` 角色推荐
- `docs/模型选型建议.md` — 人类可读说明

**公式**：`G_doc = 0.50×V + 0.30×A + 0.20×C`（申报填表）；无视觉则不计算 G。通用 `G = 0.30×V + 0.35×A + 0.35×C`。

## 三、L0：离线命令与套件

在项目根目录执行：

```bash
python smoke_test_models.py --offline
```

**通过标准**：末尾输出 `[OK] offline 检查通过`，退出码为 0。

**子套件顺序**（实现见 `_run_all_offline`）：

1. `ContentGenerator` 路由（Mock 向量库：强命中 / 空库联网 / 空库关联网 / 低相似联网）
2. `query_expander.expand_query`：长度上限、同义词扩展
3. `_max_output_tokens`：表格 / 短段 / 长段与硬顶
4. `WordFiller.clean_table_answer`
5. `WordFiller` 段落占位与摘要章节相关离线回归（含 `filler_abstract_instruction`）
6. 创新点表格扫槽与括号槽位回归

**不再属于一期默认门禁**：审核辅助、`rule_audit` / `need_model_audit`、`task_grouper`、`evidence_planner`、`prepare_bundle_from_evidence` 路由对齐、`batch_generator` JSON 解析、`template_vision`、多模态表格、`docx_typography`。

排障：设置环境变量 `SMOKE_VERBOSE=1` 可打印异常栈。

## 六、L1 / L2 手工抽检清单

1. **联网**：空库或低相似段，`content_gen_route` 中 `native_web_search=True`；强命中且未触发弱库/低相似时 `native_web_search=False`。
2. **模板**：上传带 `{{ANCHOR_NAME}}` 的 `.docx` 模板，分析后应识别到对应 FillTask；无锚点模板应得到一期范围提示，而不是继续走默认视觉分析路径。
3. **Word**：打开 `_已填写.docx` 无报错；主要锚点已替换；抽查正文和表格格无明显 Markdown 残留。
4. **知识库切换**：更换活动知识库后，同 query 检索片段来源或内容应有变化。
5. **人工复核定位**：生成完成后的界面提示应强调“初稿生成完成，可下载后继续人工复核”。
