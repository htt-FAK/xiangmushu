# 测试与验收说明

本文档与 [smoke_test_models.py](../smoke_test_models.py) 的 `--offline` 套件对齐，描述分层测试策略、模块用例矩阵及手工验收要点。

**与外部 GPT 优化计划文档的差异**：仓库未实现 `AUDIT_MODE`、`ENABLE_TASK_GROUPING` 等独立环境变量开关；审核为侧栏「启用审核 Agent」+ 代码内规则优先逻辑；分组预检索在 `app.py` 生成流程中默认执行（无单独「分组 RAG」侧栏开关）。以 [config.py](../config.py) 与 `app.py` 为准。

## 一、测试分层

| 层级 | 命令 / 方式 | 依赖 | 目的 |
|------|-------------|------|------|
| L0 离线 | `python smoke_test_models.py --offline` | 无 API Key | 路由、检索扩展、分组、证据压缩、bundle 对齐、审核规则、表格清洗、JSON 围栏解析等纯逻辑 |
| L1 API 冒烟 | `python smoke_test_models.py`（可加 `--skip-vision`、`--skip-chroma`、`--skip-embed-api`） | 有效 Key | 小/大模型、视觉、Embedding、Chroma 入库、流式；并再次跑完整 L0 套件 |
| L1 模型矩阵 | `python smoke_test_models.py --probe-models` | 网关 Key | 13 模型 chat/vision/enable_search → `data/probe_gateway_models.md` |
| L1 评分榜 | `python scripts/run_leaderboard.py --quick` 或 `python smoke_test_models.py --leaderboard --leaderboard-quick` | 网关 + 百炼 Key | V/A/C/G + 速度 → `docs/模型评分榜.md`、`data/leaderboard/model_selection.yaml` |
| L2 端到端 | `streamlit run app.py`，实际上传模板与知识库并生成 | Key + 本地数据 | UI、Word 输出、侧栏开关与日志是否符合预期 |

## 二、L1：模型评分榜（V/A/C/G + 速度）

配置：[data/leaderboard/leaderboard_v2026.05.21.yaml](../data/leaderboard/leaderboard_v2026.05.21.yaml)

```bash
# 骨架自检（不调 API）
python scripts/run_leaderboard.py --dry-run

# 推荐：quick 子集 × 复星网关 + 百炼双通道
python scripts/run_leaderboard.py --quick --channels fosun,dashscope

# 指定模型
python scripts/run_leaderboard.py --models qwen3.5-plus,gpt-5.4 --channels fosun,dashscope

# 仅 C 榜离线（smoke --offline 映射到 repo_level）
python scripts/run_leaderboard.py --offline-c-only

# 经冒烟入口
python smoke_test_models.py --leaderboard --leaderboard-quick
```

**产出**：

- `docs/模型评分榜.md` — 质量主榜、G 榜、低延迟副榜、E 榜
- `data/leaderboard/results_<timestamp>.json` — 原始分数
- `data/leaderboard/model_selection.yaml` — 各 `config.py` 角色推荐
- `docs/模型选型建议.md` — 人类可读说明

**公式**：`G_doc = 0.50×V + 0.30×A + 0.20×C`（申报填表）；无视觉则不计算 G。通用 `G = 0.30×V + 0.35×A + 0.35×C`。

## 三、L0：离线命令与套件

在项目根目录执行：

```bash
python smoke_test_models.py --offline
```

**通过标准**：末尾输出 `[OK] offline 检查通过`，退出码为 0。

**子套件顺序**（实现见 `_run_all_offline`）：

1. `ContentGenerator` 路由（Mock 向量库：强命中 / 空库联网 / 空库关联网 / 低相似联网）
2. 审核辅助：`effective_word_cap`、`should_apply_revision`
3. `query_expander.expand_query`：长度上限、同义词扩展
4. `rule_audit`、`need_model_audit`
5. `_max_output_tokens`：表格 / 短段 / 长段与硬顶
6. `task_grouper.group_tasks`：同行合并、段落按章合并
7. `evidence_planner`：`retrieve_for_group`、`compress_evidence`、`format_evidence`
8. `prepare_bundle_from_evidence` 与 `_build_chat_request` 的 **model / enable_search / generation_tier / use_small_llm_for_rag** 对齐；表格任务 user 含「表格填写任务」
9. `WordFiller.clean_table_answer`
10. `batch_generator._strip_json_fence` + `json` 解析

排障：设置环境变量 `SMOKE_VERBOSE=1` 可打印异常栈。

## 五、侧栏 Session 键与行为（L2）

| Session 键 | 含义 |
|--------------|------|
| `adv_use_tavily` | 联网补料（弱库或低相似时 `enable_search`） |
| `adv_use_stream` | 流式输出 |
| `adv_use_audit_agent` | 启用审核：先 `rule_audit`，再按需调用审核模型 |
| `adv_audit_regenerate` | `major_issue` 时用审核意见重生成一次 |
| `adv_use_batch_table` | 表格行批量 JSON 生成，失败降级逐格 |

**预检索**：每次生成前对 `group_tasks` 结果调用 `retrieve_for_group` 构建 `evidence_map`，与侧栏「分组」开关无关（若日后增加开关，需同步更新本文档与离线用例）。

## 六、L1 / L2 手工抽检清单

1. **联网**：空库或低相似段，`content_gen_route` 中 `native_web_search=True`；强命中且未触发弱库/低相似时 `native_web_search=False`。
2. **审核**：勾选「启用审核 Agent」，普通短段且规则通过时应出现「规则审核通过（跳过模型审核）」类提示；故意输出含「以下是」前缀的草稿（若可构造）应触发规则问题并可能进入模型审核。
3. **批量表格**：勾选「表格行批量生成」，含多列表格模板生成时，部分任务应显示「批量生成（表格行）」；取消勾选后应全部为逐格路由提示。
4. **Word**：打开 `_已填写.docx` 无报错；抽查表格格无异常换行堆积、无过长 Markdown 符号。
5. **知识库切换**：更换活动知识库后，同 query 检索片段来源或内容应有变化。
6. **表格列语义（宽表）**：列数 ≥5 时批量表格应**降级**为逐格（日志含 `wide table` 或不再出现「批量生成（表格行）」）；生成内容不应把整行左侧「问题」抄进「是/否」短答列。
7. **表头短答列**：表头含「是/否」「页码」「片段」等时，对应格应为**极短短语**（或「资料未载明」），不应出现整段工作流/方案叙述；回填后无「资料N：____」占位骨架残留。
