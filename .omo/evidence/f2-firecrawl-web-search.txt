  FINDING-04 — `GenerationBundle.__getattr__` (L197-207):
    **DEAD CODE.** Grep the repo for production callers of the convenience
    API — there are none. All production code uses the dataclass fields
    directly (bundle.route_meta, bundle.model, bundle.temperature,
    bundle.extra_body, bundle.messages, bundle.evidence_refs,
    bundle.ref_texts). All tests also use bundle.route_meta["…"].
    Added "for convenience" during migration, but no caller exercises it.

    Worse: it's an active smell. A `__getattr__` that forwards into a dict
    shadows attribute semantics — if a test later adds a route_meta key
    that collides with a real dataclass field (e.g. "model"), the lookup
    order becomes order-dependent on Python's own __getattribute__ vs
    __getattr__ machinery and silently masks bugs.

    VERDICT: unjustified mutation. Remove the method. If a caller ever
    wants route_meta shorthand, they should just do `bundle.route_meta["…"]`.
    Keep the dataclass as the explicit record.

  FINDING-05 — route_meta `native_web_search` key:
    • L607 hardcodes False.
    • L806 reads `extra_body.get("enable_search")` — but extra_body never
      has that key, so it's also always False.
    • server.py L826 bundle.route_meta.get("native_web_search") — always False.
    • content_auditor.py L113 / L306 — always False.
    • document_models.py GenerationTrace.native_web_search — dead bool.
