KEY DESIGN DECISIONS
--------------------
- fetch_web_evidence (from web_search_agent.py) is called — NOT search_web_evidence directly.
  This preserves cache reuse via self._web_cache and centralizes the Firecrawl contract.

- Firecrawl injection happens BEFORE user_msg formatting (after vision_block computation,
  before USER_PROMPT_PARA/USER_PROMPT_TABLE.format()), so the augmented ref_texts flows
  into {retrieved_texts} correctly for both paragraph and table_cell tasks.

- The use_plus branch now uses the SAME generation model as the default (else) branch,
  just with tier="main_writer_web_evidence" for observability. No model switching, no
  extra_body enable_search.

- Error resilience: if fetch_web_evidence raises or returns error/empty facts, generation
  proceeds without the 【联网证据】 block — no exception propagates.

- table_cell handling is untouched: _fast_table_cell_route and table_cell_mm logic unchanged.
  The extra_body.pop("enable_search", None) in fast_mode branch is a harmless no-op
  (use_plus=False always in fast_mode, so enable_search was never set).
