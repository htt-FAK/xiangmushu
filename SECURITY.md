# 安全策略 / Security Policy

## 支持的版本 / Supported Versions

我们仅对最新稳定版本提供安全更新。建议所有用户保持使用最新发行版。

We provide security updates for the latest stable release only. We recommend all users stay on the latest release.

| 版本 / Version | 安全更新 / Security Updates |
| --- | --- |
| 最新 main 分支 / Latest `main` | ✅ |
| 旧版本 / Older releases | ❌ |

---

## 响应流程 / Response Process

1. 我们将在 **48 小时内** 确认收到你的报告。
   We will acknowledge receipt of your report within **48 hours**.

2. 我们会评估漏洞的严重程度并告知你预计的处理时间。
   We will assess the severity and provide an estimated timeline for remediation.

3. 修复完成后，我们会发布安全公告并感谢你负责任的披露。
   Once a fix is released, we will publish a security advisory and credit your responsible disclosure.

---

## 负责任披露 / Responsible Disclosure

在漏洞修复发布之前，请勿公开披露相关信息。我们会与你协作，确保问题得到及时、妥善的处理。

Please do not disclose the vulnerability publicly until a fix has been released. We will work with you to ensure timely and responsible resolution.

感谢你的帮助，让我们一起让项目更安全。

Thank you for helping us keep the project safe.
