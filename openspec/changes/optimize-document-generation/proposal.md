## Why

当前 Word 文档生成存在三个严重问题：**水印丢失**、**内容不足**、**表格格式混乱**。这些问题导致生成的文档无法直接使用，需要大量人工修复，严重影响用户体验和工作效率。

此外，现有系统缺乏**生成后质量验证机制**，无法自动检测文档的视觉质量和内容完整性。同时，引入 **MiMo-V2.5-Pro** 模型（支持联网搜索）可提升生成质量和审核能力。

- 水印丢失：python-docx 库在保存文档时不会完整保留 header/footer 元素
- 内容不足：默认字数设置偏低，模型生成内容往往过于简短
- 表格格式混乱：单元格内容替换时删除了所有格式元素
- 缺乏视觉审核：无法自动检测生成文档的视觉效果
- 缺乏二轮优化：发现问题后无法自动修复

## What Changes

### 基础修复
- **水印保留机制**：在文档处理前后提取并恢复水印 XML 元素，确保页眉/页脚中的水印图片和文字不丢失
- **内容充实度优化**：
  - 提高默认字数设置（快速 300→500，普通 500→800，增强 800→1200）
  - 优化生成提示词，强调内容充实
  - 添加内容充实度审核规则
- **表格格式保留**：修改单元格内容替换逻辑，只替换文本内容，保留原有格式元素（字体、颜色、边框等）

### 视觉审核与二轮优化
- **视觉审核模块**：将生成的 Word 文档转为图片，使用 VLM (qwen3.6-plus-2026-04-02) 进行质量评估
  - 水印完整性评分
  - 格式正确性评分
  - 内容充实度评分
  - 表格规范性评分
  - 排版美观度评分
- **二轮优化机制**：基于视觉审核结果，自动诊断问题并重新生成问题段落
  - 最多 3 轮优化循环
  - 每轮记录改进情况
  - 综合评分决定是否通过

### MiMo 模型集成
- **MiMo API 配置**：添加 MiMo-V2.5-Pro 模型支持（支持联网搜索，2026-05-30 过期）
- **模型路由**：视觉审核和大模型生成可选择使用 MiMo 或现有模型

## Capabilities

### New Capabilities

- `watermark-preservation`: 水印保留机制，在文档处理过程中保留页眉/页脚中的水印元素
- `content-richness`: 内容充实度控制，确保生成内容达到预期字数和质量
- `table-format-preservation`: 表格格式保留，在单元格内容替换时保留原有格式
- `visual-audit`: 视觉审核，使用 VLM 评估生成文档的视觉质量
- `document-optimization`: 文档优化，基于审核结果进行二轮优化
- `mimo-integration`: MiMo 模型集成，支持使用 MiMo-V2.5-Pro 进行生成和审核

### Modified Capabilities

- `chapter-paragraph-fill`: 修改段落填充行为，增加内容充实度检查
- `hello-page`: 更新首页显示的默认字数参数说明

## Impact

**核心文件变更**：
- `core/filler.py`：添加水印保留逻辑，修改单元格内容替换方法
- `core/docx_typography.py`：扩展格式保留功能
- `core/generator.py`：优化生成提示词，集成 MiMo 模型
- `core/content_auditor.py`：添加内容充实度审核规则
- `core/visual_auditor.py`：新增视觉审核模块
- `core/document_optimizer.py`：新增文档优化模块
- `config.py`：添加 MiMo API 配置和视觉审核配置
- `app.py`：调整默认字数参数，添加视觉审核开关

**向后兼容性**：
- 所有变更向后兼容，不影响现有 API 接口
- 默认参数调整可通过环境变量覆盖
- 视觉审核可通过环境变量禁用

**依赖影响**：
- 无新增外部依赖
- 需要 MiMo API Key（2026-05-30 过期）
- 可能需要更精细的 python-docx XML 操作
