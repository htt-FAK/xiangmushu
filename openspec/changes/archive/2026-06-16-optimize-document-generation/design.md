## Context

当前 `WordFiller` 类（`core/filler.py`）负责将 AI 生成的内容回填到 Word 模板。存在以下问题：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          当前架构问题                                     │
└─────────────────────────────────────────────────────────────────────────┘

问题 1: 水印丢失
───────────────────────────────────────────────────────────────────────────
  原始文档                    filler.py 处理流程                 输出文档
┌──────────────┐            ┌──────────────────────┐          ┌──────────────┐
│  水印        │            │                      │          │  水印丢失!   │
│  (header/    │ ─────────▶ │  Document(path)     │ ───────▶ │              │
│   footer)    │            │  doc.save(output)    │          │              │
└──────────────┘            └──────────────────────┘          └──────────────┘
                            
原因: python-docx 在保存时不会完整保留 header/footer 中的水印元素

问题 2: 内容不足
───────────────────────────────────────────────────────────────────────────
  界面设置                     生成流程                         输出内容
┌──────────────┐            ┌──────────────────────┐          ┌──────────────┐

问题 3: 表格格式混乱
───────────────────────────────────────────────────────────────────────────
  原始单元格                   filler.py 处理                    输出单元格
┌──────────────────┐        ┌──────────────────────┐          ┌──────────────┐

└──────────────────┘        └──────────────────────┘          └──────────────┘
```

## Goals / Non-Goals

**Goals:**
1. **水印保留**：文档处理后保留原有水印（包括图片水印和文字水印）
2. **内容充实**：生成内容达到预期字数，避免过于简短
3. **格式保留**：表格单元格内容替换时保留原有格式（字体、颜色、边框）
4. **视觉审核**：使用 VLM 自动评估生成文档的视觉质量
5. **二轮优化**：基于审核结果自动修复问题，最多 3 轮
6. **MiMo 集成**：支持使用 MiMo-V2.5-Pro 进行生成和审核

**Non-Goals:**
- 不修改 python-docx 库本身
- 不改变现有的模板分析逻辑
- 不影响现有的段落填充逻辑
- 不处理页眉/页脚中的其他内容（只关注水印）
- 视觉审核不替代人工最终审核（仅作辅助）

## Decisions

### 决策 1: 水印保留方案

**选择方案**: XML 提取-恢复机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     水印保留流程                                          │
└─────────────────────────────────────────────────────────────────────────┘

  ┌─────────────┐      ┌─────────────────┐      ┌─────────────────┐
  │ 原始文档    │      │ 提取水印 XML    │      │ 处理文档内容    │
  │             │ ───▶ │ (header/footer) │ ───▶ │ (填充内容)      │
  └─────────────┘      └─────────────────┘      └─────────────────┘
                                                      │
                                                      ▼
  ┌─────────────┐      ┌─────────────────┐      ┌─────────────────┐
  │ 最终文档    │ ◀─── │ 恢复水印 XML    │ ◀─── │ 保存文档        │
  │ (保留水印)  │      │                 │      │                 │
  └─────────────┘      └─────────────────┘      └─────────────────┘
```

**实现方式**：
```python
class WatermarkPreserver:
    """水印保留器：提取、保存、恢复水印元素"""
    
    @staticmethod
    def extract_watermarks(doc: Document) -> dict:
        """从文档中提取所有水印 XML 元素"""
        
    @staticmethod  
    def restore_watermarks(doc: Document, watermarks: dict) -> None:
        """将水印 XML 元素恢复到文档中"""
```

**备选方案**：
- 方案 A: 使用 docxtpl 库（放弃：需要重构整个填充逻辑）
- 方案 B: 使用 LibreOffice 命令行转换（放弃：性能开销大）
- 方案 C: XML 提取-恢复（选择：最小侵入性，可控）

### 决策 3: 表格格式保留

**选择方案**: 精细化单元格内容替换

**当前代码问题** (`filler.py:570-575`):
```python
def _clear_cell_body_keep_tcPr(cell) -> None:
    tc = cell._tc
    for child in list(tc):
        if child.tag == qn("w:tcPr"):
            continue
        tc.remove(child)  # 删除所有内容！
```

**改进方案**:
```python
def _replace_cell_text_preserve_format(cell, new_text: str) -> None:
    """替换单元格文本但保留格式"""
    for para in cell.paragraphs:
        if para.runs:
            # 保留第一个 run 的格式
            first_run = para.runs[0]
            rPr = deepcopy(first_run._r.rPr)
            # 清空所有 run
            for run in para.runs:
                run.text = ""
            # 在第一个 run 写入新文本
            first_run.text = new_text
            if rPr is not None:
                first_run._r.insert(0, rPr)
        else:
            para.add_run(new_text)
```

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 水印 XML 结构复杂，不同 Word 版本可能有差异 | 水印恢复失败 | 添加异常捕获，失败时记录日志但不中断流程 |
| 内容字数增加可能导致 token 消耗增加 | 成本上升 | 通过环境变量控制是否启用，提供配置选项 |
| 表格格式保留可能不适用于所有场景 | 部分格式丢失 | 提供 `PRESERVE_TABLE_FORMAT` 环境变量控制开关 |
| 修改核心填充逻辑可能引入新 bug | 稳定性风险 | 添加单元测试，保持向后兼容 |

### 决策 6: MiMo 模型集成

**选择方案**: 新增客户端配置，保持现有路由逻辑

```python
# config.py
MIMO_API_KEY = os.getenv("MIMO_API_KEY", "")
MIMO_BASE_URL = "https://api.xiaomimimo.com/v1"
MIMO_MODEL = "mimo-v2.5-pro"

def mimo_client() -> OpenAI:
    """MiMo API 客户端"""
    return OpenAI(api_key=MIMO_API_KEY, base_url=MIMO_BASE_URL)
```

**使用场景**：
- 视觉审核：可选择使用 MiMo 或百炼
- 内容生成：可选择使用 MiMo 或 DeepSeek
- 联网搜索：MiMo 支持内置搜索

## Migration Plan

**阶段 1: 基础修复（低风险）**
1. 新增 `WatermarkPreserver` 类
2. 在 `fill_template()` 中集成水印提取/恢复
3. 添加配置开关 `PRESERVE_WATERMARK`
4. 调整 `app.py` 中的默认字数参数
5. 优化 `generator.py` 中的提示词

**阶段 2: MiMo 集成（低风险）**
1. 在 `config.py` 添加 MiMo API 配置
2. 新增 `mimo_client()` 函数
3. 在生成器中添加 MiMo 模型路由

**阶段 3: 视觉审核（中风险）**
1. 新增 `core/visual_auditor.py` 模块
2. 实现 docx→PDF→PNG 转换
3. 实现 VLM 视觉评分
4. 集成到生成流程

**阶段 4: 二轮优化（中风险）**
1. 新增 `core/document_optimizer.py` 模块
2. 实现问题诊断逻辑
3. 实现段落级重新生成
4. 实现优化循环（最多3轮）

**回滚策略**：
- 每个阶段独立，可通过环境变量关闭
- 保留原有方法作为 fallback
- MiMo 失败自动回落到百炼/DeepSeek
